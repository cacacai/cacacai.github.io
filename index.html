发表于 2019-03-03 | 评论数：

在Java web项目结构如下图情况下


在src的java文件中需要在Webcontent中生成一些下载类型文件时候，在不同运行状态下的路径都不一样

比如在测试单元运行时候获取到的是当前项目的实际路径比如F:\workspace\bastionhub；
但是在debug或者在打包成war在真实服务器中运行的时候，获取到的路径是运行时tomcat的路径，

在struts中可以通过以下方法来获取运行项目的绝对路径，在需要生成文件的时候直接使用绝对路径来屏蔽不同的运行状态影响去相对的获取。
比如

Struts1 获取相对路径的方法：

String upload_filepath =

this.getServlet().getServletContext().getRealPath(“/“) ;

Struts2 获取相对路径的方法：

String upload_filepath =new File(ServletActionContext.getServletContext().getRealPath(“/“);
