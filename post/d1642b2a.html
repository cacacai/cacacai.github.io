Capture::Tiny

提供了一种简单，可移植的方式来捕获发送到STDOUT(标准输出)或STDERR(标准错误输出)的几乎任何内容，无论它是来自Perl，来自XS代码还是来自外部程序。可选地，输出可以激活捕捉，以便在传递到原始文件句柄时捕获它。因此，它甚至适用于Windows（通常）。不用去想在什么情况下用什么用什么捕获模块，用这个就够了。
https://metacpan.org/pod/Capture::Tiny

DBI

DBI是Perl编程语言的数据库访问模块。它定义了一组方法，变量和约定，它们提供了一致的数据库接口，与所使用的实际数据库无关。重要的是要记住DBI只是一个接口。DBI是应用程序与一个或多个数据库驱动程序模块之间的“粘合剂”层。它是执行大部分实际工作的驱动程序模块。DBI为驱动程序提供了标准的接口和框架。
DBI架构

https://metacpan.org/pod/DBI

IO::All

IO :: All将所有最好的Perl IO模块组合成一个漂亮的面向对象的界面，大大简化了日常的Perl IO习语。它导出一个名为io的函数，它返回一个新的IO :: All对象。那个对象可以做到这一切！IO :: All对象是IO :: File，IO :: Dir，IO :: Socket，Tie :: File，File :: Spec，File :: Path，File :: MimeInfo和File :: ReadBackwards的代理;以及所有DBM和MLDBM模块。您可以使用这些类中的大多数方法以及IO :: Handle（它们从中继承）。IO :: All添加了许多其他有用的惯用方法，包括文件统计和操作函数。IO :: All是可插拔的，IO :: All :: LWP和IO :: All :: Mailto等模块增加了更多功能。可选地，每个IO :: All对象都可以绑定到自身。这意味着您可以在其上使用大多数perl IO内置：readline，<>，getc，print，printf，syswrite，sysread，close。

Storable

Storable包为包含SCALAR，ARRAY，HASH或REF对象的Perl数据结构带来持久性，即可以方便地存储到磁盘并在以后检索的任何内容。它可以通过调用store来引用要存储的对象以及应该写入图像的文件名，以常规的过程方式使用它。例程对I / O问题或其他内部错误返回undef，否则返回true值。严重错误会作为骰子异常传播。要检索存储到磁盘的数据，请使用带有文件名的检索。存储在该文件中的对象将重新创建到内存中，并返回对根对象的引用。如果在读取时发生I / O错误，则返回undef。其他严重错误通过die传播。由于存储是以递归方式执行的，因此您可能希望将对共享大量公共数据的对象的引用填充到单个数组或哈希表中，然后存储该对象。这样，当您检索整个事物时，对象将继续共享它们最初共享的内容。
